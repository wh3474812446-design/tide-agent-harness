# 工具与权限模型

## 工具定义

Tide 中的每个工具都需要声明：

```ts
interface Tool {
  name: string;
  description: string;
  inputSchema: JsonSchema;
  risk: "read" | "write" | "execute" | "network";
  concurrencySafe: boolean;
  execute(input, context): Promise<string>;
}
```

模型只看到工具名称、描述和输入 Schema。风险等级、并发策略和实际执行函数只由 Tide 运行时掌握。

## 当前内置工具

| 工具 | 风险 | 用途 |
|---|---|---|
| `list_files` | `read` | 列出工作区内文件和目录 |
| `read_file` | `read` | 读取 UTF-8 文本文件 |
| `write_file` | `write` | 写入或覆盖文件 |
| `replace_in_file` | `write` | 替换文件中的指定文本 |
| `create_directory` | `write` | 新建文件夹 |
| `copy_path` | `write` | 复制文件或文件夹 |
| `move_path` | `write` | 移动或重命名文件和文件夹 |
| `delete_path` | `write` | 删除文件或文件夹 |
| `run_command` | `execute` | 在工作区执行命令 |

## 风险等级

| 等级 | 含义 | 示例 |
|---|---|---|
| `read` | 只读取本地数据 | 列目录、读文件 |
| `write` | 修改本地文件或状态 | 写文件、移动文件、删除文件夹 |
| `execute` | 执行系统命令 | npm、git、脚本运行 |
| `network` | 访问网络资源 | HTTP API 工具 |

## 授权配置

`.env` 中通过 `HARNESS_ALLOW_RISKS` 配置允许的风险等级：

```text
HARNESS_ALLOW_RISKS=read,write,network
```

当前配置允许 Tide 读取文件、管理工作区文件夹并调用 HTTP API。`execute` 默认不放开，避免模型直接运行命令。

## 路径边界

所有本地文件工具都会先解析路径，再确认目标仍然位于 Tide 工作区内。

会被拒绝的路径示例：

```text
../secret.txt
C:\Users\OtherProject\config.json
```

额外保护：

- 不允许删除整个工作区根目录。
- 不允许移动整个工作区根目录。
- 覆盖目标为工作区根目录时会拒绝。

## 生产建议

在更高风险环境中，建议继续补充：

- 写入目录白名单。
- 删除操作回收站或备份机制。
- API 域名白名单。
- 命令执行沙箱。
- 高风险操作二次确认。
- 工具执行审计日志。
