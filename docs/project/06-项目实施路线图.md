# 项目实施路线图

## 当前版本能力

Tide 当前已经具备可运行的本地 Agent Harness 基础能力：

- 浏览器中文界面。
- DeepSeek 等国产模型配置。
- OpenAI-compatible 模型适配。
- 本地文件读写和文件夹管理工具。
- JSON API 工具配置。
- 权限风险策略。
- 会话持久化。
- 自动化测试覆盖核心链路。

## 近期迭代

建议优先完善以下功能：

| 优先级 | 功能 | 价值 |
|---|---|---|
| P0 | 文件管理可视化面板 | 让用户直观看到 Tide 管理的文件 |
| P0 | API 工具配置编辑器 | 在网页里新增、测试和保存 API 工具 |
| P1 | 权限开关界面 | 不改 `.env` 即可调整 `read/write/network` |
| P1 | 会话列表 | 支持查看、切换、删除历史会话 |
| P1 | 工具执行详情 | 展示参数、输出、错误和耗时 |

## 中期迭代

中期目标是从本地工具项目升级为可配置工作台：

- 知识库上传和检索。
- 工作区文件树。
- 工具执行审计。
- 多模型对比。
- API 工具模板市场。
- 对接 MCP 工具。
- 支持长任务和中断恢复。

## 生产化迭代

如果 Tide 需要用于真实业务环境，应补充：

- 用户身份和访问控制。
- 加密密钥管理。
- 写入操作备份。
- 网络域名白名单。
- 命令执行沙箱。
- 结构化日志和监控。
- 错误告警。
- 自动化部署脚本。

## 技术债管理

当前可以接受的技术债：

- 会话存储仍为本地文件。
- Web UI 仍为原生 HTML/CSS/JS。
- API 工具配置仍通过 JSON 文件加载。
- 工具权限以风险等级为主，还没有细粒度资源策略。

需要避免扩大的技术债：

- 在 Agent Loop 中堆业务逻辑。
- 绕过 Tool Executor 直接执行工具。
- 在前端保存明文密钥到 localStorage。
- 把模型供应商差异泄漏到核心循环。
