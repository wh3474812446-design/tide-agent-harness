# 生产化清单

## 安全

- [ ] `.env` 不进入版本控制。
- [ ] API Key 不写入前端 localStorage。
- [ ] 文件工具限制在工作区内。
- [ ] 写入工具配置目录白名单。
- [ ] 删除操作支持备份或回收站。
- [ ] 命令执行默认关闭。
- [ ] 网络工具配置域名白名单。
- [ ] 高风险操作二次确认。

## 权限

- [ ] 区分普通用户和管理员。
- [ ] 按会话限制可用工具。
- [ ] 按目录限制文件访问。
- [ ] 按 API 工具限制调用权限。
- [ ] 记录所有高风险工具调用。

## 可观测性

- [ ] 记录模型请求耗时。
- [ ] 记录工具调用耗时。
- [ ] 记录工具错误原因。
- [ ] 记录上下文裁剪次数。
- [ ] 记录 API 工具状态码。
- [ ] 提供运行日志导出。

## 稳定性

- [ ] 模型 API 超时和重试。
- [ ] 网络工具超时和重试。
- [ ] 长任务中断恢复。
- [ ] 会话写入失败恢复。
- [ ] 大文件读取限制。
- [ ] 工具输出落盘与摘要回填。

## 数据管理

- [ ] 会话保留周期。
- [ ] 会话删除能力。
- [ ] 敏感信息脱敏。
- [ ] 密钥轮换流程。
- [ ] 备份和恢复策略。

## 发布

- [ ] 明确 Node.js 版本要求。
- [ ] 固定依赖版本。
- [ ] 提供一键启动脚本。
- [ ] 提供配置模板。
- [ ] 发布前运行 `npm run check`。
- [ ] 记录版本变更。
