# Tide 项目总览

## 项目定位

Tide 是一个本地 Agent Harness 项目，用于把大模型、工具调用、权限策略、上下文管理、会话存储和浏览器界面组合成可运行的智能体系统。

项目目标不是提供单一聊天壳，而是提供一个可扩展的本地执行底座：

- 连接 DeepSeek、Qwen、GLM、MiniMax、Kimi、小米 MiMo、Anthropic 和 OpenAI-compatible 模型。
- 让模型通过受控工具读取、写入和管理项目文件。
- 将 HTTP API 配置成模型可调用的业务工具。
- 在本地保留会话、事件和配置，便于二次开发。
- 通过浏览器界面完成聊天、模型切换、API Key 配置和运行状态查看。

## 核心能力

| 能力域 | 当前实现 |
|---|---|
| 模型接入 | OpenAI-compatible 适配器、Anthropic 适配器、脚本 Provider |
| 本地工具 | 列目录、读文件、写文件、替换文件、新建目录、复制路径、移动路径、删除路径、运行命令 |
| API 工具 | 通过 JSON 配置 HTTP API，并暴露为模型工具 |
| 权限策略 | 按 `read`、`write`、`execute`、`network` 风险等级授权 |
| 上下文 | 消息裁剪、工具调用与工具结果配对保留 |
| 会话 | 本地 `.sessions/` 持久化 |
| Web UI | 中文深色界面、模型配置面板、工具状态、事件流、聊天输入 |

## 项目边界

Tide 是本地项目，不是云端多租户平台。默认设计假设：

- 用户在自己的电脑上运行 Tide。
- `.env` 保存本地模型和 API 配置。
- 工具只在项目工作区内处理文件。
- 高风险命令执行能力需要显式放开。

Tide 不默认提供：

- 云端账号体系。
- 多用户权限隔离。
- 远程沙箱。
- 生产级审计后台。
- 自动支付、计费或企业级配额管理。

## 推荐使用场景

- 搭建自己的本地 AI 助手。
- 用国产模型调用本地文件工具和业务 API。
- 快速验证 Agent 工具协议、权限策略和上下文机制。
- 为后续 MCP、插件、知识库、自动化工作流做底层基础。

## 关键目录

```text
src/
  app/         Tide 运行时和模型配置
  core/        Agent 主循环
  model/       模型 Provider 适配
  tools/       本地工具与 API 工具
  policy/      权限策略
  context/     上下文管理
  session/     会话存储
web/           浏览器可视化界面
docs/project/  项目说明文档
examples/      API 工具示例
tests/         自动化测试
```

## 运行入口

```powershell
npm run web:open
```

或双击桌面快捷方式：

```text
Open Tide Web.cmd
```

命令行聊天入口：

```powershell
npm start
```
