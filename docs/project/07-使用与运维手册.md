# 使用与运维手册

## 安装依赖

```powershell
npm install
```

## 启动浏览器界面

```powershell
npm run web:open
```

也可以双击桌面快捷方式：

```text
Open Tide Web.cmd
```

如果修改了 `.env` 中的权限、模型或 API 工具配置，需要重启服务后生效。

## 启动命令行聊天

```powershell
npm start
```

一次性任务：

```powershell
npm run chat -- "请读取 README.md 并总结项目能力"
```

## 模型配置

浏览器界面内置模型配置面板，可以切换：

- DeepSeek
- Qwen
- GLM
- MiniMax
- Kimi
- 小米 MiMo
- Anthropic
- OpenAI-compatible

配置保存到 `.env`，真实 API Key 不应提交到公开仓库。

## 文件管理能力

当前 `.env` 已允许：

```text
HARNESS_ALLOW_RISKS=read,write,network
```

因此 Tide 可以在工作区内：

- 查看文件列表。
- 读取文件。
- 写入文件。
- 替换文件内容。
- 新建文件夹。
- 复制文件或文件夹。
- 移动或重命名。
- 删除文件或文件夹。

文件工具不会越过 Tide 项目目录。

## API 工具

默认 API 工具配置：

```text
HARNESS_API_TOOLS=examples/api-tools.example.json
```

修改 API 工具 JSON 后需要重启服务。

## 常见问题

### 网页打不开

确认服务是否启动，或重新运行：

```powershell
npm run web:open
```

### 模型没有响应

检查：

- `.env` 中 API Key 是否正确。
- Base URL 是否正确。
- 模型名称是否可用。
- 账户是否有余额或调用权限。

### 工具被拒绝

检查 `.env` 中是否允许对应风险等级：

```text
HARNESS_ALLOW_RISKS=read,write,network
```

### 文件夹管理失败

确认目标路径在 Tide 项目目录内，并且不是整个工作区根目录。
