# 架构决策记录

## ADR-001：本地优先

Tide 选择本地优先架构，配置、会话和工具执行默认发生在用户电脑上。

原因：

- 降低部署门槛。
- 方便接入本地文件系统。
- 便于快速迭代模型和工具。
- 避免过早引入多租户复杂度。

影响：

- 需要用户自己保护 `.env`。
- 生产多用户场景需要额外服务端设计。

## ADR-002：核心循环保持模型无关

Agent Loop 不直接依赖某个模型厂商协议，而是通过 Provider 接口交互。

原因：

- 国产模型和海外模型都能接入。
- OpenAI-compatible 服务可以复用同一适配器。
- 后续替换模型不会影响工具执行链路。

影响：

- Provider 层需要处理各厂商工具调用格式差异。

## ADR-003：工具统一经过 Registry 和 Executor

所有工具必须注册到 Tool Registry，并通过 Tool Executor 执行。

原因：

- 统一参数校验。
- 统一权限判断。
- 统一输出截断。
- 统一事件记录。

影响：

- 不应在业务代码中绕过 Executor 直接执行高风险能力。

## ADR-004：权限按风险等级分层

当前权限模型使用 `read`、`write`、`execute`、`network` 四类风险等级。

原因：

- 简洁清晰，适合本地 Harness。
- 能覆盖文件、命令和网络工具的主要风险。
- `.env` 配置易理解。

影响：

- 细粒度权限需要在后续版本中补充资源级策略。

## ADR-005：API 工具使用 JSON 配置

Tide 允许通过 JSON 把 HTTP API 配置为模型工具。

原因：

- 用户可以不写代码接入业务 API。
- API Key 可通过环境变量模板注入。
- JSON Schema 可以限制模型输入。

影响：

- 复杂认证和流式任务仍建议写 TypeScript 工具。

## ADR-006：Web UI 采用原生前端

当前 Web UI 使用原生 HTML、CSS 和 JavaScript。

原因：

- 项目轻量。
- 启动成本低。
- 方便直接部署为静态资源加 Node 服务。

影响：

- 当界面复杂度继续提升时，可以迁移到 React、Vue 或 Svelte。
